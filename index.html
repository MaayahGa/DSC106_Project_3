Write-up

The project aims to show how Alaskaâ€™s average monthly and seasonal temperatures have changed over time in comparison with other U.S. states, using climate model data. In order to help users quickly recognize long-term warming patterns present in Alaska, we have implemented the following designs in our visualization:

Bar Charts & Trend Line: We chose a diverging bar chart to represent the monthly differences in average temperature. Bar height makes it easy to compare positive vs. negative changes from the baseline, which is the average of monthly temperatures across all years. Users can immediately see which months were warmer or cooler. We added a button below the viz so that users can see clarification of how each bar is calculated, while not being abrupt. Additionally, we also added a subtle dotted trend line to help users recognize the long-term trend of warming without overwhelming the main chart.

Color Palette: We used a bipolar redâ€“blue color scheme, where red indicates temperatures warmer than average and blue indicates temperatures cooler than average. These colors are intuitive and widely used in climate-related visualizations. We also have considered using color saturation differences to help highlight differences in magnitude, but it was rejected for distracting from the directional meaning of temperature change. The dotted trend lines are either yellow or green to make it contrast with red and blue without implying a third temperature category.

Navigation & Filter Buttons: The tabs across the top allow users to switch between states. This supports comparison across multiple geographies while maintaining consistency in visual format. In these regional visualizations we included the regression line of Alaska to make comparison easier. The month and season filter buttons enable users to isolate patterns within a specific time period. This function can be helpful in testing hypotheses like â€œAre winters warming faster than summers?â€ We also have considered dropdowns, but the tab format made comparisons faster and reduced the number of clicks. In addition, we include icons next to state names and seasons that provide visual identity. By default, we opted for â€œAll Seasonsâ€ and â€œAll Monthsâ€ to provide a big-picture perspective before users apply filters.

Hover on Bars: The visualization omits some information to keep it clean and avoid overwhelming users, such as months, absolute temperatures and precise magnitude of difference in temperature from the baseline. Including all of these directly on the chart created visual clutter during early iterations and made the chart harder to read at a glance. Instead, we surface these details through hover tooltips, so that precise values are revealed only when the user expresses interest.

Visual Style: We used a card-based interface with soft edges to give the visualization an accessible, modern feel. This improves readability and separates content sections clearly, such that the UI is not feeling cluttered despite numerous filter options.
